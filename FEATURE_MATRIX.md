- **Now** – Required for the first usable version.
- **Next** – Planned after the core experience is complete.
- **Later** – Long-term ideas.

# Success Criteria

The MVP is considered successful when:

- It becomes the first application opened every morning.
- It reduces context switching between multiple applications.
- The Home page answers what needs attention within five seconds.
- New widgets can be added without changing the core architecture.

---

# Guiding Principle

Build only what provides immediate value.

Every new feature should answer one question:

**"Will I use this regularly?"**

If the answer is no, it belongs in **Later**.
